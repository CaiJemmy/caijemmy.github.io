软链接和硬链接

软链接

软链接是一个特殊的独立文件，它的内容是目标文件的路径字符串

  • 软链接有自己的inode和文件类型l。
  • 访问软链接时，系统会读取其内容，然后跳转到目标路径。

创建与删除

1ln -s 目标路径 软连接名
  • 删除软链接本身：rm softlink，不影响目标
  • 删除目标文件，软链接变成“悬空链接”（指向不存在的路径），访问时报错。

特点与限制

  • 可以跨文件系统（因为只存储路径字符串）
  • 可以对目录创建软链接
  • 可以指向不存在的路径（不检查目标是否存在）
  • 软链接的权限（lrwxrwxrwx）是固定的，实际访问时取决于目标文件的权限
  • 占用少量空间（存储路径字符串）

硬链接

硬链接是多个文件名指向同一个inode

  • 创建硬链接后，新文件名与旧文件名共享相同的inode编号
  • 硬链接计数会增加

创建与删除

1ln 源文件 硬链接名

删除任意一个文件名，只要还有至少一个文件名指向该inode，文件数据就不会被真正删除。只有当硬链接计数为0时，磁盘空间才被释放。

特点与限制

  • **不能跨文件系统：**因为inode编号只在各自文件系统内唯一
  • **不能对目录创建硬链接：**防止循环引用，虽然系统底层允许但普通用户禁止
  • 修改任意一个链接，所有链接看到的内容都相同（因为共享数据块）
  • 不占用额外空间（只增加一条目录项，约几个字节）
  • 硬链接文件与源文件的文件类型、权限、所有者、时间戳完全一致

对比表格

特性 硬链接 软链接
inode 与源文件相同 自己的独立inode
文件类型 普通文件 符号链接
跨文件系统 不可以 可以
指向目录 不可以 可以
目标删除后 数据仍存在 链接失效
大小 仅目录项 存储路径字符串的大小
权限 与源文件完全相同 固定0777，实际由目标决定
相对路径 不支持 支持
查找所有链接 通过find -samefile 或 inode号 无法直接反向查找

使用场景

硬链接适用场景

  • 备份/快照 节省空间
  • 程序文件版本管理（多个名字指向同一可执行文件）
  • 需要多个路径访问同一份数据，且不关心文件是否被“移动”

软链接适用场景

  • 快捷方式（跨分区、跨文件系统）
  • 动态库版本管理
  • 链接到目录
  • 需要指向可能变化的路径

注意

  • 硬链接 != 复制：复制是新的inode + 新的数据块，硬链接共享数据块

  • 软链接相对路径问题：相对路径是相对于软链接所在目录解析，而不是当前工作目录

  • 删除原始文件

    • 硬链接：数据还在，其他链接可用
    • 软链接：变成断链

  • 目录硬链接：除非使用mount --bind，否则不要尝试。系统不允许，因为会导致目录循环和rm -fr 无限递归。
