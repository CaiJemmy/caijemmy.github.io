iptables filter表基本命令

  • iptables和netfilter的关系
  • iptables的5个链

    • 给本机某个进程的数据包：PREROUTING –> INPUT
    • 由本机转发的数据包：PREROUTING –> FORWARD –> POSTROUTING
    • 由本机发出的数据包：OUTPUT –> POSTROUTING
  • 表和链的关系

    • 表是按功能分
    • 链是按时机分
    • 链容纳各种防火墙规则
    • 链将规则按照时机来分类

注意：老的系统（如CentOS7）中，nat还会出现在INPUT，更新系统已经不会了。

iptables的5个表

表名 主要用途 内核模块
filter 包过滤（防火墙核心），确认是否放行 iptables_filter
nat 网络地址转换（DNAT/SNAT/MASQUERADE） iptables_nat
mangle 修改数据包头（TTL、TOS、MARK等），拆解、修改、重新封装数据包 iptables_mangle
raw 绕过连接跟踪（NOTRACK），是否对数据包进行状态跟踪 iptables_raw
security 强制访问控制（SELinux 相关，较少使用） iptables_security

指定匹配条件

通用匹配

  • 可直接使用，不依赖于其他条件或扩展
  • 包括网络协议、IP地址、网络接口等条件

隐含匹配

  • 要求以特定的协议匹配作为前提
  • 包括端口、TCP标记、ICMP类型等条件

显示匹配/扩展匹配

  • 要求以"-m 扩展模块"的形式明确指出类型
  • 包括多端口、MAC地址、IP范围、数据包状态等条件

指定目标

1-j, --jump target

filter表中的target可以是：

  • 内置的目标
    • ACCEPT：允许通过
    • DROP：丢弃数据包
    • RETURN：停止遍历此链，继续前一个调用链中的下一个规则
  • 用户自定义链
  • 扩展目标：
    • 参加 man iptables-extensions

警告

为防止ssh登录不上，建议配置前设置个定时器，清理配置，保证自己对主机的连接权。

​
 1(cat << 'EOF'
 2#!/bin/bash
 3. /etc/profile
 4LOGFILE="/var/log/iptables_rescue.log"
 5TIMESTAMP=$(date "+%Y-%m-%d %H:%M:%S")
 6check_ssh_in_rule() {
 7    total_dpt=$(iptables -L INPUT -n 2>/dev/null | grep -c "dpt:22")
 8    [ "$total_dpt" != "0" ] && { echo "[$TIMESTAMP] dpt:22 in rule check ok" >> "$LOGFILE"; return; }
 9    iptables -I INPUT 1 -p tcp --dport 22 -j ACCEPT 2>/dev/null && echo "[$TIMESTAMP] 规则已恢复 in rule" >> "$LOGFILE" || echo "[$TIMESTAMP] 规则恢复 in rule 失败" >> "$LOGFILE"
10}
11check_ssh_out_rule() {
12    total_spt=$(iptables -L OUTPUT -n 2>/dev/null | grep -c "spt:22")
13    [ "$total_spt" != "0" ] && { echo "[$TIMESTAMP] spt:22 out rule check ok" >> "$LOGFILE"; return; }
14    iptables -I OUTPUT 1 -p tcp --sport 22 -j ACCEPT 2>/dev/null && echo "[$TIMESTAMP] 规则已恢复 out rule" >> "$LOGFILE" || echo "[$TIMESTAMP] 规则恢复 out rule 失败" >> "$LOGFILE"
15}
16check_ssh_port() {
17    if systemctl is-active ssh >/dev/null 2>&1 || systemctl is-active sshd >/dev/null 2>&1 || ss -tln 2>/dev/null | grep -q ":22 "; then
18        echo "[$TIMESTAMP] ssh 服务正常启动" >> "$LOGFILE"
19        return 0
20    fi
21    echo "[$TIMESTAMP] ssh 服务未启动，请检查服务" >> "$LOGFILE"
22    return 1
23}
24echo "-------------------------- [$TIMESTAMP] begin--------------------------" >> "$LOGFILE"
25check_ssh_in_rule
26check_ssh_out_rule
27check_ssh_port
28echo "-------------------------- [$TIMESTAMP] end--------------------------" >> "$LOGFILE"
29EOF
30) > /usr/local/bin/iptables_rescue.sh && chmod +x /usr/local/bin/iptables_rescue.sh && touch /var/log/iptables_rescue.log && chmod 644 /var/log/iptables_rescue.log && (crontab -l 2>/dev/null | grep -v "/usr/local/bin/iptables_rescue.sh"; echo "*/2 * * * * /usr/local/bin/iptables_rescue.sh") | crontab - && echo "安装成功！"
​
