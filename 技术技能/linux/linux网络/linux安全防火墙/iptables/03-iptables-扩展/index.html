iptables 扩展

iptables 的匹配扩展（match extensions）是其强大功能的核心，它们通过 -m <module> 语法加载，用于在规则中实现更精细、更语义化的数据包匹配（如匹配端口范围、连接状态、字符串、IP 地址类型、速率、应用层特征等）。这些扩展由内核模块（xt_*）提供，需对应模块已加载（如 xt_tcp, xt_conntrack, xt_string 等）。

以下为 iptables（基于 Linux 内核 4.x–6.x 主流版本，兼容 iptables-nft 和 legacy 模式）中常用且稳定的核心匹配扩展的详细分类列表。

📌 注：所有扩展均作用于 filter、nat、mangle、raw、security 表中支持该扩展的链（如 INPUT、OUTPUT、FORWARD、PREROUTING、POSTROUTING），但并非所有扩展在所有表/链中都可用（例如 DNAT/SNAT 仅在 nat 表中有效，而 string 仅在 filter 表的 INPUT/FORWARD/OUTPUT 中常用）。

基础协议相关扩展（无需 -m，隐式加载）

这些是协议内置匹配项，不需显式 -m tcp 等，但实际由对应 xt_* 模块支撑：

协议 隐式匹配项 说明 示例
tcp --sport, --dport, --tcp-flags, --syn 等 匹配 TCP 头字段 iptables -A INPUT -p tcp --dport 22 -j ACCEPT
udp --sport, --dport 匹配 UDP 端口 iptables -A INPUT -p udp --dport 53 -j ACCEPT
icmp --icmp-type 匹配 ICMP 类型（如 echo-request） iptables -A INPUT -p icmp --icmp-type 8 -j ACCEPT
icmpv6 --icmpv6-type IPv6 ICMP 类型匹配 ip6tables -A INPUT -p icmpv6 --icmpv6-type 128 -j ACCEPT

提示：-p tcp 自动加载 xt_tcp 模块；同理 -p udp → xt_udp，-p icmp → xt_icmp。

核心通用匹配扩展（最常用）

模块名 (-m) 功能说明 典型选项与示例 注意事项
conntrack
（推荐替代旧 state）		 基于连接跟踪（conntrack）匹配连接状态、原始/回复方向、NAT 信息等 --ctstate ESTABLISHED,RELATED
--ctorigsrc 192.168.1.100
--ctdir ORIGINAL		 ✅ 强烈推荐替代已废弃的 state 模块；需 nf_conntrack 内核支持；iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
multiport 匹配多个端口或端口范围（TCP/UDP），解决 --dport 只能单端口限制 --dports 22,80,443
--sports 1024:65535
--ports 53（用于 UDP/TCP 同端口）		 ⚠️ 最多支持 15 个端口；仅支持 TCP/UDP；
iptables -A INPUT -p tcp -m multiport --dports 22,80,443 -j ACCEPT
iprange 按IP 地址范围（非 CIDR）匹配源/目的 IP --src-range 192.168.1.10-192.168.1.50
--dst-range 203.0.113.1-203.0.113.254		 ✅ 灵活于 CIDR（如需连续但非网段的 IP）；
iptables -A INPUT -m iprange --src-range 10.0.0.5-10.0.0.20 -j DROP
mac 匹配源 MAC 地址（仅链路层有效，INPUT/FORWARD 有效） --mac-source 00:11:22:33:44:55 ⚠️ 仅对本地接收/转发的帧有效；无法匹配经路由转发后的 MAC（因 MAC 在每跳重写）；
iptables -A INPUT -m mac --mac-source aa:bb:cc:dd:ee:ff -j ACCEPT
owner
（仅 OUTPUT 链）		 匹配本地发起连接的进程 UID/GID/PID/进程名等 --uid-owner 1000
--gid-owner www-data
--pid-owner 1234
--cmd-owner wget		 ⚠️ 仅 OUTPUT 链可用；需 CONFIG_NETFILTER_XT_MATCH_OWNER=y；普通用户进程 UID 匹配需 root；
iptables -A OUTPUT -m owner --uid-owner nginx -j REJECT

🔹 三、网络层与连接行为扩展

模块名 功能说明 示例 注意事项
addrtype 匹配地址类型（如 UNICAST, LOCAL, BROADCAST, ANYCAST, MULTICAST, UNSPEC） --src-type LOCAL（匹配本机地址）
--dst-type UNICAST		 ✅ 常用于区分本机 vs 外部流量；iptables -A OUTPUT -m addrtype --src-type LOCAL -j ACCEPT
pkttype 匹配数据包类型（unicast, broadcast, multicast） --pkt-type broadcast 仅对 INPUT/PREROUTING 有效；
iptables -A INPUT -m pkttype --pkt-type broadcast -j DROP
physdev
（桥接专用）		 匹配桥接设备上的物理入/出接口（--physdev-in, --physdev-out, --physdev-is-bridged） --physdev-in eth0 ⚠️ 仅用于 Linux bridge（如 br0）环境；需 CONFIG_NETFILTER_XT_MATCH_PHYSDEV=y；
iptables -A FORWARD -m physdev --physdev-in eth0 --physdev-out br0 -j ACCEPT
pktowner
（较新）		 类似 owner，但支持 --socket-exists（匹配已有 socket 的包） --socket-exists 内核 ≥ 4.15；用于更精确的本地进程控制；

🔹 四、高级匹配与应用层扩展（需谨慎启用）

模块名 功能说明 示例 注意事项
string 深度包检测（DPI）：匹配数据包载荷（payload）中的 ASCII/HEX 字符串 --algo bm（Boyer-Moore）
--from 40（从第 40 字节开始）
--string "GET /admin"		 ⚠️ 性能开销大，慎用于高流量链路；需 CONFIG_NETFILTER_XT_MATCH_STRING=y；
iptables -A FORWARD -p tcp --dport 80 -m string --algo bm --string "sqlmap" -j DROP
time 基于系统时间/日期匹配（支持周几、月份、时间范围） --timestart 09:00
--timestop 17:00
