虚拟网络设备

veth 对

veth 对的基本概念

veth（Virtual Ethernet Device） 是 Linux 内核提供的一种虚拟网络设备，总是成对出现：

  • 一端在宿主机，通常命名为 vethxxxx
  • 另一端在容器的 network namespace 中，通常命名为 eth0
  • 这对设备像一根虚拟的网线，将两个网络命名空间连接起来

veth 对的创建和连接过程

手动创建 veth 对的步骤：

 1# 1. 创建 veth 对（一端叫 veth0，另一端叫 peer0）
 2sudo ip link add veth0 type veth peer name peer0
 3
 4# 2. 查看创建的网络设备
 5ip link show | grep veth
 6
 7# 3. 创建一个新的网络命名空间（模拟容器）
 8sudo ip netns add testns
 9
10# 4. 将 peer0 移动到新的命名空间
11sudo ip link set peer0 netns testns
12
13# 5. 在宿主机中配置 veth0
14sudo ip addr add 10.0.0.1/24 dev veth0
15sudo ip link set veth0 up
16
17# 6. 在新的命名空间中配置 peer0（现在它是容器的 eth0）
18sudo ip netns exec testns ip addr add 10.0.0.2/24 dev peer0
19sudo ip netns exec testns ip link set peer0 up
20sudo ip netns exec testns ip link set lo up
21
22# 7. 验证连接
23ping -c 2 10.0.0.2

Docker 容器的实际创建过程：

当 Docker 创建一个容器时：

1# Docker 内部执行的类似命令：
2# 1. 创建 veth 对
3ip link add veth123456 type veth peer name eth0
4
5# 2. 将 eth0 移动到容器的 network namespace
6ip link set eth0 netns <container-pid>
7
8# 3. 将 veth123456 连接到 docker0 网桥
9brctl addif docker0 veth123456

3. 删除宿主机 vethxxx 的影响

直接回答：会断开！

删除宿主机端的 vethxxx 会发生：

  1. 立即断开连接：

    1# 删除宿主机端的 veth 设备
2sudo ip link delete veth123456
    • 容器内的 eth0 会立即变成 NO-CARRIER 状态
    • 网络连接完全中断

  2. 容器内的表现：

    1# 在容器内查看网络状态
2ip link show eth0
3# 输出：eth0: <NO-CARRIER,BROADCAST,MULTICAST,UP> ...

  3. 恢复方法：

    • 重启容器：Docker 会自动重建 veth 对
    • 手动重建：需要重新创建 veth 对并重新连接

重要特性：

  • veth 对是双向连接的
  • 删除任何一端，另一端都会失效
  • 但另一端不会自动删除，只是变为不可用状态

4. 多 veth 连接问题分析

问题 1：一个 eth0 连接多个 vethxxx？

答案：技术上可以，但不推荐也不常见

1# 理论上可以创建多个 veth 对连接到同一个命名空间
2sudo ip link add veth1 type veth peer name eth1
3sudo ip link add veth2 type veth peer name eth2
4sudo ip link set eth1 netns testns
5sudo ip link set eth2 netns testns
6
7# 在容器内会有 eth1 和 eth2 两个接口

但 Docker 的标准做法是：

  • 每个容器默认只有一个 eth0
  • 如果需要多个网络接口，使用 --network 参数： 
    1docker run --network=bridge --network=my-network ...

问题 2：宿主机多个 vethxxx 连接到容器？

答案：可以，这就是多网卡容器的实现方式

 1# Docker 创建多网络容器
 2docker network create net1
 3docker network create net2
 4
 5# 容器连接到多个网络
 6docker run -d --name multi-net \
 7  --network=bridge \
 8  --network=net1 \
 9  --network=net2 \
10  nginx
11
12# 查看结果
13docker exec multi-net ip link show
14# 会有：eth0@ifxxx, eth1@ifyyy, eth2@ifzzz 等多个接口

每个网络连接都会创建一对 veth：

  • 宿主机：vethA → 容器：eth0（默认 bridge）
  • 宿主机：vethB → 容器：eth1（net1）
  • 宿主机：vethC → 容器：eth2（net2）

6. 实际示例和操作

查看 Docker 容器的 veth 连接：

 1# 1. 查看容器进程 ID
 2docker inspect <container-id> | grep Pid
 3# "Pid": 12345
 4
 5# 2. 查看容器的网络命名空间
 6sudo ls -la /proc/12345/ns/
 7# lrwxrwxrwx ... net -> net:[4026531234]
 8
 9# 3. 查看容器内的网络接口
10sudo nsenter -t 12345 -n ip link show
11# 会看到 eth0@ifxxx
12
13# 4. 在宿主机找到对应的 veth
14ip link show | grep ifxxx
15# veth123456@ifxxx
16
17# 5. 查看 veth 对
18ethtool -S veth123456
19# 显示配对信息

删除和恢复实验：

 1# 实验：删除 veth 对的影响
 2# 1. 运行一个测试容器
 3docker run -d --name test --rm alpine sleep 3600
 4
 5# 2. 找到对应的 veth
 6VETH=$(ip link show | grep $(docker inspect test | grep SandboxKey | cut -d'"' -f4) | cut -d' ' -f2 | cut -d'@' -f1)
 7
 8# 3. 删除 veth
 9sudo ip link delete $VETH
10
11# 4. 查看容器网络状态
12docker exec test ip link show eth0
13# 显示 NO-CARRIER
14
15# 5. 重启容器网络
16docker restart test
17# veth 对会自动重建

7. 总结要点

  1. veth 对是成对的：一端在宿主机，一端在容器
  2. 删除一端即断开：删除宿主机 veth，容器 eth0 会失去连接
  3. 多连接是可能的：
    • 一个容器可以有多个网络接口（eth0, eth1, …）
    • 每个接口对应一对 veth
    • 宿主机有多个 veth 连接到不同容器
  4. Docker 网络模型：
    • 默认：容器 eth0 ↔ vethxxx ↔ docker0 网桥
    • 多网络：每个网络创建一个新的 veth 对
  5. 自动管理：Docker 会自动管理 veth 对的生命周期

最佳实践：

  • 不要手动删除 Docker 管理的 veth 设备
  • 使用 Docker 命令管理网络（docker network）
  • 需要多网络时，使用 Docker 网络功能而非手动配置
  • 调试时使用 docker network inspect 查看连接关系
