01 Mysql主从复制

一主一从架构搭建

主从术语演变

数据库 旧术语 新术语 版本
MySQL Master/Slave Source/Replica 8.0+

主节点环境准备

注意主机到主节点容器的映射目录和端口

1docker run  -d -p 3307:3306 --privileged=true \
2-v /app/mysql/source/log:/var/log/mysql \
3-v /app/mysql/source/data:/var/lib/mysql \
4-v /app/mysql/source/conf:/etc/mysql/conf.d \
5-e MYSQL_ROOT_PASSWORD=123456 \
6--name mysql_source mysql:8.4.8

主节点my.cnf 配置

 1[mysqld]
 2# [必须]主服务器唯一ID
 3server-id=1
 4
 5# [必须]启用二进制日志,指名路径。比如：自己本地的路径/log/mysqlbin
 6log-bin=mysql-bin
 7
 8# [可选] 0（默认）表示读写（主机），1表示只读（从机）
 9read-only=0
10
11# [可选]设置不要复制的数据库
12binlog-ignore-db=test,mysql
13
14# [可选]设置需要复制的数据库,默认全部记录。比如：binlog-do-db=my_test
15binlog-do-db=my_test
16
17# [可选]设置binlog格式
18binlog_format=STATEMENT
19
20# 设置日志文件保留的时长，单位是秒
21binlog_expire_logs_seconds=6000
22
23# 控制单个二进制日志大小。此参数的最大和默认值是1GB
24max_binlog_size=200M
25
26# 设置二进制日志使用内存大小（事务）
27binlog_cache_size=1M
28
29# 二进制日志过期清理时间，默认为0，表示不自动清理
30# 5.7 及以前	expire_logs_days	唯一选项
31# 8.0.0 - 8.0.28	expire_logs_days 和 binlog_expire_logs_seconds	两者共存，建议使用新的
32# 8.0.29+	binlog_expire_logs_seconds	expire_logs_days 被弃用
33# 8.4.0+	binlog_expire_logs_seconds	expire_logs_days 被移除
34# expire_logs_days=7
35binlog_expire_logs_seconds=604800
36
37# 跳过主从复制中遇到的所有错误或指定类型的错误，避免slave端复制终端
38# 如：1062错误是指一些主键重复，1032错误是因为从数据库数据不一致
39slave_skip_errors=1062

主节点mysql数据库中配置

 1# 1. 配置文件完成后，先删除之前的容器，重新启动一个新的容器。
 2docker rm -f $(docker ps --filter "name=mysql_source" --filter "status=running" -q)
 3
 4docker run  -d -p 3307:3306 --privileged=true \
 5-v /app/mysql/source/log:/var/log/mysql \
 6-v /app/mysql/source/data:/var/lib/mysql \
 7-v /app/mysql/source/conf:/etc/mysql/conf.d \
 8-e MYSQL_ROOT_PASSWORD=123456 \
 9--name mysql_source mysql:8.4.8
10
11# 2. 在数据库中创建同步用户并赋权限
12docker exec mysql_source mysql -u root -p123456 -e "
13CREATE USER IF NOT EXISTS 'replica'@'%' IDENTIFIED BY '123456';
14GRANT REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'replica'@'%';
15"
警告

配置后不要在主节点创建数据库等操作。

从节点环境准备

注意主机到从节点容器的映射目录和端口

1docker run  -d -p 3308:3306 --privileged=true \
2-v /app/mysql/replica1/log:/var/log/mysql \
3-v /app/mysql/replica1/data:/var/lib/mysql \
4-v /app/mysql/replica1/conf:/etc/mysql/conf.d \
5-e MYSQL_ROOT_PASSWORD=123456 \

重启从节点mysql数据库

1# 1. 配置文件完成后，先删除之前的容器，重新启动一个新的容器。
2docker restart $(docker ps --filter "name=mysql_replica1" --filter "status=running" -q)
3docker ps -a

从节点配置需要复制的主节点

查看主节点的状态，并记录下File和Position的值。

1docker exec mysql_source mysql -u root -p123456 -e "
2SHOW BINARY LOG STATUS;
3"
  • 这里有个坑，之前的命令SHOW MASTER STATUS;和SHOW SOURCE STATUS;都不能用，执行HELP SHOW;命令查看所有的show命令。
​
1-- 命令模板，这个是8.4版本以上的模板，和之前的模板关键字完全不同。
2CHANGE REPLICATION SOURCE TO
3SOURCE_HOST='主机的IP地址或宿主机的IP地址',
4SOURCE_USER='复制用户名',
5SOURCE_PASSWORD='复制用户的密码',
6SOURCE_PORT=3307,
7SOURCE_LOG_FILE='mysql-bin.具体数字',
8SOURCE_LOG_POS=具体值,
9SOURCE_CONNECT_RETRY=30;
MySQL 版本 旧语法 新语法 状态
MySQL 5.7 CHANGE MASTER TO CHANGE MASTER TO 正常
MySQL 8.0 CHANGE MASTER TO CHANGE REPLICATION SOURCE TO 混合支持
MySQL 8.4 已弃用 CHANGE REPLICATION SOURCE TO 推荐使用新语法

配置命令，注意执行的是从节点mysql库。

 1docker exec mysql_replica1 mysql -u root -p123456 -e "
 2CHANGE REPLICATION SOURCE TO
 3SOURCE_HOST='10.0.0.16',
 4SOURCE_USER='replica',
 5SOURCE_PASSWORD='123456',
 6SOURCE_PORT=3307,
 7SOURCE_LOG_FILE='mysql-bin.000003',
 8SOURCE_LOG_POS=158,
 9SOURCE_CONNECT_RETRY=30;
10"

从节点查看主从复制的状态

参数值如果是NO，说明未开启。

1docker exec mysql_replica1 mysql -u root -p123456 -e "
2SHOW REPLICA STATUS \G
3" | grep -E 'Replica_IO_Running|Replica_SQL_Running'

从节点开启主从复制

 1# 1. 开启主从复制
 2docker exec mysql_replica1 mysql -u root -p123456 -e "
 3start replica;
 4SHOW REPLICA STATUS \G
 5"
 6
 7# 解决问题时先停主从复制
 8docker exec mysql_replica1 mysql -u root -p123456 -e "
 9stop replica;
10"
11
12docker exec mysql_replica1 mysql -u root -p123456 -e "
13reset replica;
14"

启动报错解决方法：

 1Last_IO_Error: Error connecting to source 'replica@10.0.0.16:3307'. This was attempt 1/10, with a delay of 30 seconds between attempts. Message: Authentication plugin 'caching_sha2_password' reported error: Authentication requires secure connection.
 2
 3# 获取公钥而不使用SSL
 4
 5docker exec mysql_replica1 mysql -u root -p123456 -e "
 6STOP REPLICA;
 7RESET REPLICA ALL;
 8
 9CHANGE REPLICATION SOURCE TO
10SOURCE_HOST='10.0.0.16',
11SOURCE_USER='replica',
12SOURCE_PASSWORD='123456',
13SOURCE_PORT=3307,
14SOURCE_LOG_FILE='mysql-bin.000003',
15SOURCE_LOG_POS=158,
16SOURCE_CONNECT_RETRY=30,
17SOURCE_SSL=0,
18GET_SOURCE_PUBLIC_KEY=1;
19
20START REPLICA;
21SHOW REPLICA STATUS \G
22"
23
24#

Replica_IO_Running和Replica_SQL_Running 的状态都为“YES”，说明链路打通。

验证

 1# 1. 主库建库
 2docker exec mysql_source mysql -u root -p123456 -e "
 3create database my_test;
 4" 2>&1 | grep -v "Using a password"
 5
 6# 2. 从库查询
 7docker exec mysql_replica1 mysql -u root -p123456 -ss -e "
 8SHOW REPLICA STATUS \G
 9SHOW databases;
10" 2>&1 | grep -v "Using a password"
11
12# 3. 主库建表查数据
13docker exec mysql_source mysql -u root -p123456 -e "
14use my_test;
15create table tb01(id int, name varchar(50));
16insert into tb01(id, name) values(1, 'tom');
17insert into tb01(id, name) values(2, '张三');
18" 2>&1 | grep -v "Using a password"
19
20# 4. 从库查询表数据
21docker exec mysql_replica1 mysql -u root -p123456 -e "
22use my_test;
23select * from tb01;
24" 2>&1 | grep -v "Using a password"

同步报错解决

1Last_IO_Error: Fatal error: The replica I/O thread stops because source and replica have equal MySQL server ids; these ids must be different for replication to work (or the --replicate-same-server-id option must be used on replica but this does not always make sense; please check the manual before using it).
2
3# 从节点没有配置my.cnf，重新配置后，从节点配置步骤重新执行一次后续操作。
4
5docker exec mysql_source mysql -u root -p123456 -e "
6drop database my_test;
7create database my_test;
8" 2>&1 | grep -v "Using a password"
