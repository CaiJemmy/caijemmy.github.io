01 Mysql主从复制

一主一从架构搭建

主从术语演变

数据库 旧术语 新术语 版本
MySQL Master/Slave Source/Replica 8.0+

主节点环境准备

注意主机到主节点容器的映射目录和端口

1docker run  -d -p 3307:3306 --privileged=true \
2-v /app/mysql/source/log:/var/log/mysql \
3-v /app/mysql/source/data:/var/lib/mysql \
4-v /app/mysql/source/conf:/etc/mysql/conf.d \
5-e MYSQL_ROOT_PASSWORD=123456 \
6--name mysql_source mysql:8.4.8

主节点mysql数据库中配置

 1# 1. 配置文件完成后，先删除之前的容器，重新启动一个新的容器。
 2docker rm -f $(docker ps --filter "name=mysql_source" --filter "status=running" -q)
 3
 4docker run  -d -p 3307:3306 --privileged=true \
 5-v /app/mysql/source/log:/var/log/mysql \
 6-v /app/mysql/source/data:/var/lib/mysql \
 7-v /app/mysql/source/conf:/etc/mysql/conf.d \
 8-e MYSQL_ROOT_PASSWORD=123456 \
 9--name mysql_source mysql:8.4.8
10
11# 2. 在数据库中创建同步用户并赋权限
12docker exec mysql_source mysql -u root -p123456 -e "
13CREATE USER IF NOT EXISTS 'replica'@'%' IDENTIFIED BY '123456';
14GRANT REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'replica'@'%';
15"
警告

配置后不要在主节点创建数据库等操作。

查看主节点的状态，并记录下File和Position的值。

1docker exec mysql_source mysql -u root -p123456 -e "
2SHOW BINARY LOG STATUS;
3"
  • 这里有个坑，之前的命令SHOW MASTER STATUS;和SHOW SOURCE STATUS;都不能用，执行HELP SHOW;命令查看所有的show命令。
​
1-- 命令模板，这个是8.4版本以上的模板，和之前的模板关键字完全不同。
2CHANGE REPLICATION SOURCE TO
3SOURCE_HOST='主机的IP地址或宿主机的IP地址',
4SOURCE_USER='复制用户名',
5SOURCE_PASSWORD='复制用户的密码',
6SOURCE_PORT=3307,
7SOURCE_LOG_FILE='mysql-bin.具体数字',
8SOURCE_LOG_POS=具体值,
9SOURCE_CONNECT_RETRY=30;
MySQL 版本 旧语法 新语法 状态
MySQL 5.7 CHANGE MASTER TO CHANGE MASTER TO 正常
MySQL 8.0 CHANGE MASTER TO CHANGE REPLICATION SOURCE TO 混合支持
MySQL 8.4 已弃用 CHANGE REPLICATION SOURCE TO 推荐使用新语法

配置命令，注意执行的是从节点mysql库。

 1docker exec mysql_replica1 mysql -u root -p123456 -e "
 2CHANGE REPLICATION SOURCE TO
 3SOURCE_HOST='10.0.0.16',
 4SOURCE_USER='replica',
 5SOURCE_PASSWORD='123456',
 6SOURCE_PORT=3307,
 7SOURCE_LOG_FILE='mysql-bin.000003',
 8SOURCE_LOG_POS=158,
 9SOURCE_CONNECT_RETRY=30;
10"
