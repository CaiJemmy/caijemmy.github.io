容器网络是什么

容器网络是指容器之间或非 Docker 工作负载之间连接和通信的能力。

网络模式概览

Drivers

Docker Engine has a number of network drivers, as well as the default “bridge”. On Linux, the following built-in network drivers are available:

Driver Description
bridge The default network driver.
host Remove network isolation between the container and the Docker host.
none Completely isolate a container from the host and other containers.
overlay Swarm Overlay networks connect multiple Docker daemons together.
ipvlan Connect containers to external VLANs.
macvlan Containers appear as devices on the host’s network.
模式 命令参数 描述 隔离性 性能 适用场景
bridge --network=bridge 默认模式，通过 docker0 网桥 中等 中等 开发、测试、单机部署
host --network=host 直接使用宿主机网络栈 无 最高 高性能应用、网络监控
none --network=none 无网络连接 完全 N/A 安全敏感、离线处理
container --network=container:<name> 共享其他容器的网络栈 无 高 边车模式、网络代理
overlay --network=<overlay> 跨主机容器网络 中等 中等 Swarm/K8s 集群
macvlan --network=macvlan 直接分配 MAC 地址 高 高 传统应用迁移
ipvlan --network=ipvlan 共享 MAC，不同 IP 高 高 高密度容器

bridge 模式（默认）

特点：

  • 每个容器获得独立的网络命名空间
  • 通过 docker0 虚拟网桥连接
  • 自动分配 IP（172.17.0.0/16）
  • 通过 iptables NAT 与外部通信

配置示例：

 1# 创建自定义 bridge 网络
 2docker network create --driver bridge \
 3  --subnet=192.168.100.0/24 \
 4  --gateway=192.168.100.1 \
 5  --ip-range=192.168.100.128/25 \
 6  my-bridge
 7
 8# 运行容器使用自定义 bridge
 9docker run -d --name web \
10  --network=my-bridge \
11  --ip=192.168.100.130 \
12  nginx
13
14# 查看网络配置
15docker network inspect my-bridge

网络拓扑：

1宿主机 eth0 (192.168.1.100)
2        │
3        ▼
4   docker0 (172.17.0.1/16) 或 my-bridge (192.168.100.1/24)
5        ├─ vethA → 容器1 eth0 (172.17.0.2)
6        ├─ vethB → 容器2 eth0 (172.17.0.3)
7        └─ vethC → 容器3 eth0 (192.168.100.130)

优缺点：

  • 优点：隔离性好、支持端口映射、易于管理
  • 缺点：NAT 带来性能开销、外部访问需要端口映射

host 模式

特点：

  • 容器直接使用宿主机的网络命名空间
  • 无网络隔离，共享 IP 和端口
  • 性能最高，无 NAT 开销

配置示例：

1# 使用 host 网络模式
2docker run -d --name nginx-host \
3  --network=host \
4  nginx
5
6# 查看容器网络（与宿主机相同）
7docker exec nginx-host ip addr show
8# 会显示宿主机的所有网络接口

使用场景：

 1# 1. 高性能网络应用
 2docker run --network=host redis
 3
 4# 2. 网络监控工具
 5docker run --network=host \
 6  -v /var/run/docker.sock:/var/run/docker.sock \
 7  nicolaka/netshoot
 8
 9# 3. 需要绑定低端口（<1024）的应用
10docker run --network=host myapp:80

注意事项：

  • 端口冲突：容器与宿主机应用不能使用相同端口
  • 安全性：容器可以监听宿主机的所有网络接口
  • Docker Desktop 中 host 模式有限制

优缺点：

  • 优点：零性能开销、无端口映射、直接访问
  • 缺点：无网络隔离、端口管理复杂

none 模式

特点：

  • 容器只有 loopback 接口（127.0.0.1）
  • 完全网络隔离
  • 需要手动配置网络（如果需要）

配置示例：

 1# 创建 none 网络模式的容器
 2docker run -it --rm \
 3  --network=none \
 4  --name isolated \
 5  alpine sh
 6
 7# 进入容器查看网络
 8docker exec isolated ip link show
 9# 只有：lo: <LOOPBACK,UP,LOWER_UP>
10
11# 手动添加网络（如果需要）
12# 1. 创建 veth 对
13# 2. 将一端加入容器网络命名空间

使用场景：

 1# 1. 安全敏感的数据处理
 2docker run --network=none \
 3  -v /data:/data \
 4  processing-tool
 5
 6# 2. 离线批处理作业
 7docker run --network=none \
 8  --memory=2g \
 9  batch-job
10
11# 3. 网络测试环境
12docker run --network=none \
13  --cap-add=NET_ADMIN \
14  network-tester

优缺点：

  • 优点：最高安全性、完全隔离
  • 缺点：无法直接通信、配置复杂

container 模式

特点：

  • 共享另一个容器的网络命名空间
  • 相同的 IP、端口、路由表
  • 类似 Kubernetes 的 Pod 网络模型

配置示例：

 1# 1. 创建主容器
 2docker run -d --name nginx-main \
 3  -p 8080:80 \
 4  nginx
 5
 6# 2. 创建共享网络的容器
 7docker run -it --rm \
 8  --network=container:nginx-main \
 9  --name sidecar \
10  alpine sh
11
12# 3. 验证网络共享
13# 在 sidecar 容器中：
14curl localhost  # 访问 nginx
15netstat -tlnp   # 看到 nginx 的监听端口

典型应用：Sidecar 模式

 1# 日志收集 sidecar
 2docker run -d --name app \
 3  my-application
 4
 5docker run -d --name log-collector \
 6  --network=container:app \
 7  -v /var/log/app:/logs \
 8  fluentd
 9
10# 网络代理 sidecar
11docker run -d --name webapp \
12  my-webapp:8080
13
14docker run -d --name envoy \
15  --network=container:webapp \
16  envoy-proxy

优缺点：

  • 优点：零延迟通信、共享 localhost、高效
  • 缺点：网络完全耦合、端口冲突风险

overlay 模式

特点：

  • 用于 Docker Swarm 集群
  • 跨主机的虚拟网络
  • 自动服务发现和负载均衡

配置示例：

 1# 1. 初始化 Swarm 集群
 2docker swarm init
 3
 4# 2. 创建 overlay 网络
 5docker network create --driver overlay \
 6  --attachable \
 7  --subnet=10.0.0.0/24 \
 8  my-overlay
 9
10# 3. 在 overlay 网络中创建服务
11docker service create \
12  --name web \
13  --network my-overlay \
14  --replicas 3 \
15  nginx
16
17# 4. 查看网络详情
18docker network inspect my-overlay

网络架构：

1主机A                                    主机B
2eth0: 192.168.1.101                    eth0: 192.168.1.102
3    │                                       │
4    ▼                                       ▼
5overlay网络 (10.0.0.0/24)             overlay网络 (10.0.0.0/24)
6    ├─ 容器1: 10.0.0.2                     ├─ 容器3: 10.0.0.4
7    └─ 容器2: 10.0.0.3                     └─ 容器4: 10.0.0.5
8    
9# 所有容器可以互相直接通信

VXLAN 封装：

1# 查看 VXLAN 接口
2ip link show type vxlan
3
4# 查看 overlay 网络路由
5docker exec <container> ip route show
6
7# 数据包封装过程：
8# 容器数据包 → VXLAN 封装 → 物理网络 → VXLAN 解封装 → 目标容器

优缺点：

  • 优点：跨主机通信、自动服务发现、负载均衡
  • 缺点：配置复杂、VXLAN 性能开销

macvlan 模式

特点：

  • 为容器分配独立的 MAC 地址
  • 容器像物理机一样接入物理网络
  • 无需 NAT，性能接近物理机

配置示例：

 1# 1. 创建 macvlan 网络
 2docker network create -d macvlan \
 3  --subnet=192.168.1.0/24 \
 4  --gateway=192.168.1.1 \
 5  --ip-range=192.168.1.32/28 \
 6  -o parent=eth0 \
 7  my-macvlan
 8
 9# 2. 运行容器
10docker run -d --name macvlan-container \
11  --network=my-macvlan \
12  --ip=192.168.1.35 \
13  nginx
14
15# 3. 查看 MAC 地址
16docker exec macvlan-container ip link show eth0

网络拓扑：

1物理网络交换机
2    │
3    ├─ 宿主机 eth0 (192.168.1.100) MAC: AA:BB:CC:DD:EE:FF
4    ├─ 容器1 eth0 (192.168.1.35)   MAC: 02:42:ac:11:00:02
5    └─ 容器2 eth0 (192.168.1.36)   MAC: 02:42:ac:11:00:03
6    
7# 所有设备在同一广播域

限制和注意事项：

 1# 1. 需要网卡支持混杂模式
 2sudo ip link set eth0 promisc on
 3
 4# 2. 宿主机无法直接与 macvlan 容器通信
 5# 解决方案：创建 macvlan 子接口
 6sudo ip link add macvlan-shim link eth0 type macvlan mode bridge
 7sudo ip addr add 192.168.1.200/24 dev macvlan-shim
 8sudo ip link set macvlan-shim up
 9
10# 3. 802.1Q VLAN 支持
11docker network create -d macvlan \
12  --subnet=10.0.0.0/24 \
13  -o parent=eth0.10 \
14  vlan10

优缺点：

  • 优点：性能最佳、无 NAT、直接路由
  • 缺点：需要物理网络配合、广播流量多

ipvlan 模式

特点：

  • 多个容器共享一个 MAC 地址
  • 每个容器有独立的 IP 地址
  • 更高效的网络利用率

配置示例：

 1# 1. L2 模式（类似 macvlan）
 2docker network create -d ipvlan \
 3  --subnet=192.168.1.0/24 \
 4  --gateway=192.168.1.1 \
 5  -o ipvlan_mode=l2 \
 6  -o parent=eth0 \
 7  my-ipvlan-l2
 8
 9# 2. L3 模式（路由模式）
10docker network create -d ipvlan \
11  --subnet=10.0.0.0/24 \
12  -o ipvlan_mode=l3 \
13  -o parent=eth0 \
14  my-ipvlan-l3
15
16# 3. 运行容器
17docker run -d --network=my-ipvlan-l2 \
18  --ip=192.168.1.50 \
19  nginx

ipvlan L2 vs L3：

特性 ipvlan L2 ipvlan L3
MAC 地址 共享宿主机的 共享宿主机的
广播域 同一广播域 不同广播域
ARP 需要 不需要
路由 二层交换 三层路由
性能 高 非常高

使用场景：

1# 高密度容器部署
2# 每个物理机运行数百个容器时，ipvlan 比 macvlan 更高效
3
4# 云环境中的网络策略
5# 可以与云平台的 SDN 集成

优缺点：

  • 优点：MAC 地址节约、性能高、支持 L3 路由
  • 缺点：配置复杂、某些网络设备不支持

选择指南

如何选择合适的网络模式？

  1. 开发测试环境：

    1# 使用默认 bridge 或自定义 bridge
2docker run --network=bridge myapp

  2. 生产单机部署：

    1# 考虑性能：host 或 macvlan/ipvlan
2# 考虑隔离：自定义 bridge

  3. 微服务架构：

    1# 使用自定义 bridge 网络隔离服务
2docker network create service-a
3docker network create service-b

  4. 集群部署：

    1# Docker Swarm: overlay 网络
2# Kubernetes: CNI 插件（如 Calico, Flannel）

  5. 传统应用迁移：

    1# 需要固定 IP: macvlan
2# 需要直接路由: ipvlan L3

安全建议：

  • 生产环境避免使用 host 模式
  • 为不同服务使用不同的网络
  • 限制网络访问（--network-alias、--ip）
  • 使用网络策略（如 Calico NetworkPolicy）
  • Devops
