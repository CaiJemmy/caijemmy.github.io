05 compose

docker compose官网文档

Docker Compose 是一个用于定义和运行多容器 Docker 应用程序的工具。它使用 YAML 文件来配置应用的服务、网络和卷，使得多容器应用的部署和管理变得简单高效。

Docker Compose 核心概念

什么是 Docker Compose？

Docker Compose 允许你通过一个配置文件（docker-compose.yml）来定义和运行多个相互关联的 Docker 容器。它特别适合开发、测试和单机部署场景。

核心组件

组件 描述 对应 Docker 命令
Services 容器定义，可以运行多个实例 docker run
Networks 自定义网络，连接服务 docker network create
Volumes 数据持久化存储 docker volume create
Secrets 敏感信息管理 docker secret
Configs 配置文件管理 docker config

总结：一文件（docker-compose.yml），两要素（服务（service）、工程（project））。

  • 服务：一个个应用容器实例，如：业务微服务、mysql等
  • 工程：由一组关联的业务容器组成的一个完整业务单元，在docker-compose.yml文件中定义

 3sudo chmod +x /usr/local/bin/docker-compose
 4
 5# 方法2：Docker Desktop（Windows/Mac）

 8# 方法3：pip 安装

20      - "8080:80/tcp"       # TCP 端口

22    # 端口范围
23    ports:
24      - "3000-3005:3000-3005"

4.4 数据卷配置

 1services:
 2  db:
 3    # 挂载主机目录
 4    volumes:
 5      - ./data:/var/lib/mysql           # 相对路径
 6      - /opt/data:/var/lib/mysql        # 绝对路径
 7      - ~/configs:/etc/configs:ro       # 只读挂载
 8    
 9    # 命名卷

11      - db_data:/var/lib/mysql          # 命名卷
12      - static_volume:/app/static
13    
14    # 临时卷（tmpfs）
15    tmpfs:
16      - /tmp
17      - /run
18    
19    # 卷挂载选项
20    volumes:
21      - type: volume
22        source: mydata
23        target: /data
24        volume:
25          nocopy: true  # 禁止从容器复制数据

14        condition: service_started

5. 网络配置详解

5.1 网络类型

 1networks:
 2  # 桥接网络（默认）
 3  frontend:
 4    driver: bridge
 5    driver_opts:
 6      com.docker.network.bridge.name: br-frontend
 7      com.docker.network.bridge.enable_icc: "true"
 8    
 9  # 覆盖网络（Swarm 集群）
10  cluster:
11    driver: overlay
12    attachable: true  # 允许独立容器连接
13    internal: true    # 内部网络，不连接到外部
14    
15  # Macvlan 网络
16  macvlan_net:
17    driver: macvlan
18    driver_opts:
19      parent: eth0
20      macvlan_mode: bridge
21    
22  # 自定义网络配置
23  custom:
24    driver: bridge
25    ipam:  # IP 地址管理
26      driver: default
27      config:
28        - subnet: 172.20.0.0/16
29          ip_range: 172.20.5.0/24
30          gateway: 172.20.0.1
31          aux_addresses:
32            host1: 172.20.0.2
33            host2: 172.20.0.3

5.2 外部网络

 1networks:
 2  # 使用现有网络
 3  existing:
 4    external: true
 5    name: my-existing-network
 6  
 7  # 创建新网络，如果不存在
 8  app_network:
 9    external:
10      name: app-network
