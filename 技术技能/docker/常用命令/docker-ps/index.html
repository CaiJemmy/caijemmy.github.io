Docker ps 命令详解

docker ps 是 Docker 中最常用、最重要的命令之一，用于查看和管理容器状态。

📋 基本用法

 1# 查看正在运行的容器
 2docker ps
 3
 4# 查看所有容器（包括已停止的）
 5docker ps -a
 6
 7# 查看最近创建的容器
 8docker ps -l
 9
10# 查看最后 n 个创建的容器
11docker ps -n 3
12
13# 查看容器大小
14docker ps -s
15
16# 查看完整输出（不截断）
17docker ps --no-trunc

📊 输出字段说明

1$ docker ps
2CONTAINER ID   IMAGE              COMMAND                  CREATED       STATUS       PORTS                                       NAMES
3c3f279d17e0a   ubuntu:22.04       "/bin/bash"              2 days ago    Up 2 hours   0.0.0.0:80->8080/tcp, :::80->8080/tcp       web-server
4b6f3c5d8e2f1   postgres:15        "docker-entrypoint.s…"   3 weeks ago   Up 3 weeks   0.0.0.0:5432->5432/tcp, :::5432->5432/tcp   db-primary

各字段详细说明：

字段 说明 示例
CONTAINER ID 容器唯一标识符（前12个字符） c3f279d17e0a
IMAGE 容器使用的镜像 ubuntu:22.04, nginx:latest
COMMAND 容器启动时执行的命令 "/bin/bash", "nginx -g 'daemon off;'"
CREATED 容器创建时间 2 days ago, 3 weeks ago
STATUS 容器状态 Up 2 hours（运行中）, Exited (0) 5 hours ago（已停止）
PORTS 端口映射 0.0.0.0:80->8080/tcp（主机80映射到容器8080）
NAMES 容器名称 web-server, db-primary

🎯 实用选项详解

1. 过滤容器

 1# 按状态过滤
 2docker ps --filter "status=running"
 3docker ps --filter "status=exited"
 4docker ps --filter "status=created"
 5docker ps --filter "status=paused"
 6docker ps --filter "status=restarting"
 7
 8# 按标签过滤
 9docker ps --filter "label=environment=production"
10docker ps --filter "label=app=web"
11
12# 按名称过滤
13docker ps --filter "name=web"
14docker ps --filter "name=^web.*"  # 正则表达式
15
16# 按镜像过滤
17docker ps --filter "ancestor=nginx"
18docker ps --filter "ancestor=nginx:latest"
19
20# 按退出码过滤
21docker ps -a --filter "exited=0"    # 正常退出
22docker ps -a --filter "exited=137"  # 被 SIGKILL 杀死
23docker ps -a --filter "exited!=0"   # 异常退出
24
25# 多个过滤条件组合
26docker ps --filter "status=running" --filter "label=app=api"
27
28# 过滤最近创建
29docker ps -a --filter "since=web-server"  # 在 web-server 之后创建的容器
30docker ps -a --filter "before=db"         # 在 db 之前创建的容器

2. 格式化输出

 1# 自定义输出格式
 2docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Status}}\t{{.Names}}"
 3
 4# 只显示特定字段
 5docker ps --format "{{.Names}}: {{.Status}}"
 6
 7# 显示所有字段（JSON格式）
 8docker ps --format json
 9
10# 常用格式化模板
11docker ps --format "table {{.ID}}\t{{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}"
12docker ps --format "table {{.Names}}\t{{.Status}}\t{{.RunningFor}} ago\t{{.Command}}"
13
14# Go模板高级用法
15docker ps --format '{{.ID}} {{.Names}} {{.Status}}' | column -t

3. 查看详细信息

 1# 显示完整容器ID
 2docker ps --no-trunc
 3
 4# 显示容器大小（磁盘使用）
 5docker ps -s
 6
 7# 输出字段说明：
 8# SIZE: 容器可写层的大小
 9# VIRTUAL SIZE: 容器总大小（镜像+可写层）
10
11# 安静模式（只显示容器ID）
12docker ps -q
13
14# 安静模式配合其他命令
15docker ps -aq | xargs docker stop  # 停止所有容器
16docker ps -q --filter "status=exited" | xargs docker rm  # 删除所有已停止容器

🔧 高级用法和技巧

1. 实时监控容器

 1# 实时查看容器状态（类似 top 命令）
 2docker stats
 3
 4# 持续监控容器列表（每秒刷新）
 5watch -n 1 docker ps
 6
 7# 监控特定容器
 8watch -n 1 'docker ps --filter "name=web"'
 9
10# 使用别名简化命令
11alias dps='docker ps --format "table {{.ID}}\t{{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}"'

2. 批量操作

 1# 获取所有运行中容器的ID
 2docker ps -q
 3
 4# 停止所有运行中的容器
 5docker ps -q | xargs docker stop
 6
 7# 删除所有已停止的容器
 8docker ps -aq --filter "status=exited" | xargs docker rm
 9
10# 删除所有容器（强制）
11docker ps -aq | xargs docker rm -f
12
13# 重启所有运行中的容器
14docker ps -q | xargs docker restart
15
16# 更新所有容器的重启策略
17docker ps -q | xargs docker update --restart=unless-stopped

3. 诊断和调试

 1# 查看异常退出的容器
 2docker ps -a --filter "exited!=0"
 3
 4# 查看容器的运行时长
 5docker ps --format "table {{.Names}}\t{{.Status}}\t{{.RunningFor}}"
 6
 7# 查看容器使用的端口
 8docker ps --format "table {{.Names}}\t{{.Ports}}" | grep -v "^\s*$"
 9
10# 查看容器占用的资源
11docker ps -q | xargs docker stats --no-stream
12
13# 检查容器健康状态
14docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Health}}"

4. 实用脚本示例

 1#!/bin/bash
 2# container-info.sh
 3
 4# 显示详细的容器信息
 5echo "=== 容器状态概览 ==="
 6docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}\t{{.Image}}"
 7
 8echo -e "\n=== 资源使用情况 ==="
 9docker ps -q | xargs docker stats --no-stream --format \
10  "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}\t{{.MemPerc}}\t{{.NetIO}}\t{{.BlockIO}}"
11
12echo -e "\n=== 异常容器检查 ==="
13docker ps -a --filter "exited!=0" --format \
14  "table {{.Names}}\t{{.Status}}\t{{.ExitCode}}\t{{.Command}}"
15
16echo -e "\n=== 容器数量统计 ==="
17echo "运行中: $(docker ps -q | wc -l)"
18echo "已停止: $(docker ps -aq --filter "status=exited" | wc -l)"
19echo "总数: $(docker ps -aq | wc -l)"
 1#!/bin/bash
 2# container-monitor.sh
 3
 4# 实时监控容器状态
 5watch -n 2 -c '
 6clear
 7echo "🕐 $(date)"
 8echo "=========================================="
 9echo "📦 容器状态"
10echo "=========================================="
11docker ps --format "table {{.ID}}\t{{.Names}}\t{{.Status}}\t{{.RunningFor}}\t{{.Ports}}" | tail -n +2
12echo ""
13echo "💾 资源使用"
14echo "=========================================="
15docker stats --no-stream --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}\t{{.MemPerc}}\t{{.NetIO}}"
16'

📝 常用命令组合

日常维护

 1# 清理所有已停止的容器
 2docker ps -aq --filter "status=exited" | xargs docker rm
 3
 4# 清理所有未使用的容器（包括停止的、未使用的网络、构建缓存等）
 5docker system prune -a
 6
 7# 查看容器日志
 8docker ps | grep nginx | awk '{print $1}' | xargs docker logs -f
 9
10# 进入容器
11docker ps | grep web | awk '{print $1}' | xargs -I {} docker exec -it {} bash

安全检查

1# 检查特权容器
2docker ps --filter "label=privileged=true"
3
4# 检查以 root 用户运行的容器
5docker ps --format "table {{.Names}}\t{{.Command}}" | grep "root"
6
7# 检查未限制资源的容器
8docker ps --format "table {{.Names}}\t{{.Status}}" | grep -E "(unlimited|no limit)"

网络相关

1# 查看容器使用的网络
2docker ps --format "table {{.Names}}\t{{.Networks}}"
3
4# 查看暴露的端口
5docker ps --format "table {{.Names}}\t{{.Ports}}" | grep -v "^\s*$"
6
7# 查找使用特定端口的容器
8docker ps | grep ":80->"

🎨 自定义输出格式

可用的占位符

 1# 完整列表
 2docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Command}}\t{{.CreatedAt}}\t{{.RunningFor}}\t{{.Ports}}\t{{.Status}}\t{{.Size}}\t{{.Names}}\t{{.Labels}}\t{{.Mounts}}\t{{.Networks}}"
 3
 4# 常用占位符说明：
 5.ID           容器ID
 6.Image        镜像名称
 7.Command      启动命令
 8.CreatedAt    创建时间
 9.RunningFor   运行时长
10.Ports        端口映射
11.Status       状态
12.Size         容器大小
13.Names        容器名称
14.Labels       标签
15.Mounts       挂载点
16.Networks     网络
17.Health       健康状态

美化输出示例

 1# 彩色输出（需要安装 lolcat 或使用 awk 着色）
 2docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}" | awk '
 3  BEGIN { 
 4    printf "\033[1;36m%-20s %-20s %-30s\033[0m\n", "容器名称", "状态", "端口"
 5    printf "%-20s %-20s %-30s\n", "--------------------", "--------------------", "------------------------------"
 6  }
 7  {
 8    if ($2 ~ /^Up/) 
 9      printf "\033[1;32m%-20s %-20s %-30s\033[0m\n", $1, $2, $3
10    else 
11      printf "\033[1;31m%-20s %-20s %-30s\033[0m\n", $1, $2, $3
12  }'

💡 最佳实践

  1. 使用有意义的容器名称

    1docker run --name web-app-1 nginx

  2. 为容器添加标签

    1docker run --label "environment=production" --label "app=backend" nginx

  3. 定期清理

    1# 每天清理一次
20 2 * * * docker system prune -f

  4. 使用过滤器管理容器

    1# 只操作特定环境的容器
2docker ps -q --filter "label=env=staging" | xargs docker restart

  5. 监控容器状态

    1# 设置监控脚本
2*/5 * * * * /path/to/container-monitor.sh >> /var/log/container-status.log
