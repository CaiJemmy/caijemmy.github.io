01 哈希取余算法详解

一、算法原理

1. 基本概念

哈希取余算法是一种数据分片策略，通过对数据的哈希值进行取模运算来决定数据应该分配到哪个分区/节点。

2. 算法公式

1节点编号 = hash(key) % N

其中：

  • hash(key)：对键值进行哈希运算，得到一个整数
  • N：节点总数
  • %：取模运算符
  • 结果：0 到 N-1 之间的整数，表示数据应该存储的节点编号

3. 工作流程

 1import hashlib
 2
 3def hash_mod(key, node_count):
 4    hash_obj = hashlib.md5(str(key).encode())
 5    hash_int = int(hash_obj.hexdigest(), 16)
 6
 7    return hash_int % node_count
 8
 9nodes = 3
10data_items = ["user_001", "user_002", "user_003", "user_004", "user_005"]
11
12for item in data_items:
13    node = hash_mod(item, nodes)
14    print(f"数据 '{item}' -> 节点 {node}")

4. 哈希函数选择

常见的哈希函数：

  • MD5：128位，已不推荐用于安全，但分片可用
  • SHA-1：160位
  • SHA-256：256位，更均匀
  • MurmurHash：非加密哈希，性能好
  • CRC32：简单快速

三、算法缺点

1. 节点数量变化导致大规模数据迁移

​
 1def calculate_migration(old_nodes, new_nodes, key_count=10000):
 2    """计算扩容时的数据迁移比例"""
 3    migrated = 0
 4
 5    for i in range(key_count):
 6        key = f"key_{i}"
 7        old_node = hash(key) % old_nodes
 8        new_node = hash(key) % new_nodes
 9
10        if old_node != new_node:
11            migrated += 1
12
13    migration_rate = (migrated / key_count) * 100
14    return migration_rate
15
16test_cases = [
17    (3, 4),  # 3->4 节点
18    (4, 5),  # 4->5 节点
19    (10, 11),  # 10->11 节点
20    (10, 20),  # 10->20 节点（翻倍）
21    (100, 101),  # 100->101 节点
22]
23
24print("节点扩容数据迁移比例：")
25print("-" * 40)
26for old_n, new_n in test_cases:
27    rate = calculate_migration(old_n, new_n)
28    print(f"{old_n} 节点 -> {new_n} 节点: {rate:.1f}% 数据需要迁移")

问题分析：

  • 普通取模：(N+1)/N 的数据需要重新分配
  • 3节点→4节点：约 75% 的数据需要迁移
  • 10节点→11节点：约 91% 的数据需要迁移
  • 100节点→101节点：约 99% 的数据需要迁移

3. 节点负载不均衡的连锁反应

 1# 演示一个节点故障对其他节点的影响
 2class HashModCluster:
 3    def __init__(self, node_count):
 4        self.node_count = node_count
 5        self.node_loads = [0] * node_count
 6    
 7    def add_data(self, key, data_size=1):
 8        node = hash(key) % self.node_count
 9        self.node_loads[node] += data_size
10        return node
11    
12    def node_failure(self, failed_node):
13        """模拟节点故障"""
14        print(f"\n节点 {failed_node} 故障！")
15        
16        # 故障节点的数据需要重新分配
17        failed_load = self.node_loads[failed_node]
18        self.node_loads[failed_node] = 0
19        
20        # 重新分配到其他节点（简化的重新哈希）
21        total_load_redistributed = 0
22        for i in range(failed_load):
23            # 模拟重新哈希
24            new_node = hash(f"rehash_{i}") % (self.node_count - 1)
25            if new_node >= failed_node:
26                new_node += 1
27            self.node_loads[new_node] += 1
28            total_load_redistributed += 1
29        
30        print(f"重新分配了 {total_load_redistributed} 单位负载")
31        return self.node_loads
32

四、实际应用场景

1. 适用场景

  • 场景1：静态集群，节点数固定（适合节点数基本不变的场景）
  • 场景2：无状态会话路由（会话ID路由到固定服务器）
  • 场景3：简单的负载均衡（请求分发，不涉及数据存储）

2. 不适用场景

  • 场景1：需要弹性伸缩的数据库分片
  • 场景2：需要范围查询的存储

1. 一致性哈希算法

 1# 一致性哈希解决节点变化问题
 2import bisect
 3
 4class ConsistentHash:
 5    def __init__(self, nodes, replica_count=100):
 6        self.replica_count = replica_count
 7        self.ring = {}
 8        self.sorted_keys = []
 9        
10        for node in nodes:
11            self.add_node(node)
12    
13    def add_node(self, node):
14        """添加节点，只影响部分数据"""
15        for i in range(self.replica_count):
16            key = self._hash(f"{node}:{i}")
17            self.ring[key] = node
18            bisect.insort(self.sorted_keys, key)
19    
20    def remove_node(self, node):
21        """移除节点"""
22        for i in range(self.replica_count):
23            key = self._hash(f"{node}:{i}")
24            if key in self.ring:
25                del self.ring[key]
26                self.sorted_keys.remove(key)
27    
28    def get_node(self, data_key):
29        """获取数据所在节点"""
30        if not self.ring:
31            return None
32        
33        hash_key = self._hash(data_key)
34        idx = bisect.bisect(self.sorted_keys, hash_key)
35        
36        if idx == len(self.sorted_keys):
37            idx = 0
38        
39        return self.ring[self.sorted_keys[idx]]
40    
41    def _hash(self, key):
42        """哈希函数"""
43        return hash(key) % (2**32)
44
