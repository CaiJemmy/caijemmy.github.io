iptables命令

iptables 是 Linux 内核中 netfilter 框架的用户态管理工具，其“执行过程”本质上是指数据包在内核网络协议栈中经过 netfilter 钩子（hook）时，如何被 iptables 规则匹配与处理。需注意：iptables 命令本身不“执行”数据包处理，而是配置规则；真正的匹配与动作发生在内核 netfilter 的钩子点上。以下是清晰、准确的执行过程说明（基于 IPv4，标准内核流程）：

一、数据包流经的 netfilter 钩子点（5 个关键位置）

数据包根据方向（入站/出站/转发）和协议栈阶段，依次经过以下内核 hook 点（按顺序）：

钩子点（Hook） 触发时机 对应 iptables 表/链 说明
NF_INET_PRE_ROUTING 数据包刚进入网络栈，路由决策前 raw (PREROUTING), mangle (PREROUTING), nat (PREROUTING) 可修改包头（如 DNAT）、绕过连接跟踪（raw）
NF_INET_LOCAL_IN 目标是本机的数据包，路由后、交付给上层协议前 mangle (INPUT), filter (INPUT), security (INPUT) 入站主过滤点（如允许/拒绝 SSH）
NF_INET_FORWARD 目标非本机、需转发的数据包，路由后、转发前 mangle (FORWARD), filter (FORWARD), security (FORWARD) 网关/防火墙核心转发控制点
NF_INET_LOCAL_OUT 本机生成的、发出前的数据包（路由前） raw (OUTPUT), mangle (OUTPUT), nat (OUTPUT), filter (OUTPUT) 处理本机发出的包（如 SNAT、本地端口重定向）
NF_INET_POST_ROUTING 数据包即将离开本机前（无论本地生成或转发），路由已确定 mangle (POSTROUTING), nat (POSTROUTING) 最后修改机会（如 SNAT、MASQUERADE）

注：每个 hook 点按 表优先级（raw → mangle → nat → filter → security）和 链内规则顺序（从上到下）依次匹配。

二、iptables 规则匹配与动作执行流程（单条链内）

当数据包到达某 hook 点（如 INPUT），内核按以下逻辑处理该链的规则：

  1. 顺序匹配：从链中第 1 条规则开始，逐条比对；
  2. 条件匹配：检查包的源/目的 IP、端口、协议、状态（conntrack）、接口、TCP 标志等是否满足规则条件；
  3. 匹配成功：
    • 若目标（target）为 ACCEPT / DROP / REJECT / LOG 等内置动作 → 立即终止该链匹配，执行对应动作（如丢弃或放行）；
    • 若目标为 自定义链 → 跳入该链继续匹配；返回后继续执行下一条规则（除非自定义链中明确 RETURN 或终止动作）；
    • 若目标为 SNAT / DNAT / MASQUERADE（仅在 nat 表特定链）→ 执行地址转换，并继续后续 hook 链（nat 表不终止流程，仅修改包）；
  4. 匹配失败：尝试下一条规则；
  5. 所有规则都不匹配 → 执行该链的 默认策略（policy）（如 iptables -P INPUT DROP）。

关键原则：

  • ACCEPT/DROP/REJECT 是终止动作（不继续后续规则）；
  • SNAT/DNAT 是修改动作，不终止匹配（但仅在 nat 表的 PREROUTING/OUTPUT/POSTROUTING 中生效）；
  • RETURN 用于从自定义链返回父链，继续执行父链的下一条规则。

三、典型场景示例（本机访问外网 Web 服务器）

 1本机发出请求（curl http://example.com）：
 21. 本地进程 → 协议栈 → NF_INET_LOCAL_OUT  
 3   ├─ raw OUTPUT → mangle OUTPUT → nat OUTPUT（可做源地址转换）→ filter OUTPUT  
 4   └─ 若 filter OUTPUT 允许 → 进入路由决策（查路由表）  
 52. 路由确定出口网卡（如 eth0）→ NF_INET_POST_ROUTING  
 6   └─ mangle POSTROUTING → nat POSTROUTING（如 MASQUERADE）→ 发送至网络  
 7
 8响应包返回本机：
 91. 入口网卡（eth0）→ NF_INET_PRE_ROUTING  
10   └─ raw PREROUTING → mangle PREROUTING → nat PREROUTING（DNAT）→ 连接跟踪确认为 ESTABLISHED  
112. 路由判断 → 目标是本机 → NF_INET_LOCAL_IN  
12   └─ mangle INPUT → filter INPUT（放行 ESTABLISHED）→ 交付给本地进程

四、重要补充说明

  • 状态化匹配（conntrack）：-m state --state ESTABLISHED,RELATED 或现代推荐的 -m conntrack --ctstate，依赖内核 nf_conntrack 模块，是高效防火墙的基础；
  • 规则优化建议：将高匹配频率规则（如 ESTABLISHED）放在链顶部；避免使用 -j LOG 在大量规则前（影响性能）；
  • 表与链的对应关系：
    • raw：绕过连接跟踪（NOTRACK）；
    • mangle：修改包头（TTL、TOS、MARK）；
    • nat：地址/端口转换（仅在 PREROUTING/OUTPUT/POSTROUTING）；
    • filter：核心过滤（INPUT/FORWARD/OUTPUT）；
    • security：SELinux/MAC 相关（较少使用）；
  • iptables 命令不实时处理包：iptables -A INPUT ... 仅向内核 netfilter 注册规则；包处理完全由内核在数据路径中完成，无用户态参与。
