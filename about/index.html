2、部署到互联网

2.1、创建repository

  • 建议创建两个repository，可以分别命名为MyBlog和xxx.github.io。MyBlog设置为private，保存博客源码；xxx.github.io设置为public，发布博客。

  • 在Settings > Developer Settings > Personal access tokens > tokens (classic)中创建token。注意：及时保存token名称及其序列码。

  • 在MyBlog的Settings > Secrets and variables > Actions > Repository secrets中设置token变量。

2.2、配置.gitignore

2.3、配置github的workflows

在博客主目录创建workflows文件.github\workflows\deploy.yml，拷贝以下配置内容，注意配置点。

  • 提交代码的分支
  • token配置的名称
  • blog分支名称
​
 1name: deploy
 2
 3# 代码提交到master分支时触发github action
 4on:
 5  push:
 6    branches:
 7      - master
 8
 9jobs:
10  deploy:
11    runs-on: ubuntu-latest
12    steps:
13        - name: Checkout
14          uses: actions/checkout@v4
15          with:
16              fetch-depth: 0
17
18        - name: Initialize repo and add submodule
19          working-directory: ${{ github.workspace }}
20          shell: bash
21          run: |
22            # 1) git init（一般情况下第一步可省略，此处保留以满足需求）
23            if [ -d ".git" ]; then
24              echo "Repository already initialized"
25            else
26              git init
27            fi
28            # 2) 添加子模块（若已存在则跳过）
29            if [ -d "themes/hugo-theme-relearn" ]; then
30              echo "Submodule already exists at themes/hugo-theme-relearn"
31            else
32              git submodule add -f --depth 1 https://github.com/McShelby/hugo-theme-relearn.git themes/hugo-theme-relearn
33            fi
34
35        - name: Setup Hugo
36          uses: peaceiris/actions-hugo@v3
37          with:
38              hugo-version: "latest"
39              extended: true
40
41        - name: Build Web
42          run: hugo -D
43
44        - name: Deploy Web
45          uses: peaceiris/actions-gh-pages@v4
46          with:
47              PERSONAL_TOKEN: ${{ secrets.DEV_TOKENS }}
48              EXTERNAL_REPOSITORY: xxx/xxx.github.io
49              PUBLISH_BRANCH: master
50              PUBLISH_DIR: ./public
51              commit_message: auto deploy
